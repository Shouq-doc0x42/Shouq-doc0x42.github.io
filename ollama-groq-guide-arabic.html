المقدمة — ما هي وكيف تستخدمها

ما هي واجهات برمجة الذكاء الاصطناعي (AI APIs)؟

تتيح لك واجهة برمجة الذكاء الاصطناعي إرسال نص إلى نموذج ذكاء اصطناعي مستضاف في السحابة، وتلقّي ردٍّ ذكي. تخيّلها كدالة برمجية تستدعيها عبر الإنترنت — ترسل سؤالاً أو تعليمة، يفكّر النموذج، ثم يعيد الإجابة.

أمثلة شائعة: OpenAI، Anthropic، Groq — كلها سحابية ومدفوعة حسب الاستخدام.

ما هي نماذج الذكاء الاصطناعي المحلية؟

تعمل النماذج المحلية على جهازك تماماً دون الحاجة للإنترنت. بياناتك تبقى عندك، أنت تتحكم في كل شيء، ولا رسوم على الاستخدام. أداة مثل Ollama تجعل هذا بسيطاً جداً باستخدام أمر واحد في سطر الأوامر.

خصوصية كاملة. يعمل بدون إنترنت. مجاني بالكامل.

لماذا Ollama و Groq تحديداً؟
Ollama

يشغّل النماذج مفتوحة المصدر محلياً. مجاني. البيانات لا تغادر جهازك. مثالي للسجلات الحساسة والتحقيقات الخاصة.

Groq API

استدلال سحابي فائق السرعة باستخدام معالجات LPU المتخصصة. يعالج مستندات ضخمة وإستفسارات فورية بزمن استجابة منخفض جداً.

كلاهما معاً

استخدم المحلي للبيانات الحساسة، وGroq حين تحتاج السرعة والحجم. ادمجهما في خطوط أمنية متكاملة.

💡
مثال من الواقع: محلل DFIR يستخدم Ollama محلياً لتحليل سجلات البرمجيات الخبيثة الحساسة (تبقى البيانات على الجهاز)، ثم يستخدم Groq لتلخيص تقرير حادثة يحوي 50,000 سطر في ثوانٍ معدودة.

التثبيت والإعداد — Ollama و Groq

🦙 الجزء الأول — إعداد Ollama

Ollama أداة مفتوحة المصدر تتيح لك تنزيل وتشغيل نماذج لغوية كبيرة على جهازك بأمر واحد. تُدير تنزيل النماذج وتشغّل خادم استدلال محلي على المنفذ localhost:11434.

النظام طريقة التثبيت
🍎 macOS حمّل المثبّت من ollama.com أو استخدم brew install ollama
🐧 Linux curl -fsSL https://ollama.com/install.sh | sh
🪟 Windows حمّل ملف .exe من ollama.com/download
TERMINAL — تشغيل أول نموذج سطر الأوامر
$ ollama run llama3
> Pulling model llama3... (التنزيل الأول ~4GB)
> Model loaded into memory
>>> Send a message ( /? for help)
$ ollama list
> NAME ID SIZE MODIFIED
> llama3:latest 365c0bd3... 4.7 GB 2 minutes ago
$ ollama pull mistral
> Pulling mistral:7b model...
ℹ
ماذا يحدث عند تشغيل ollama run llama3؟
يقوم Ollama بتنزيل أوزان نموذج LLaMA 3 (للمرة الأولى فقط)، يحمّلها في الذاكرة، يشغّل خادم استدلال محلي، ويفتح لك جلسة محادثة تفاعلية. التشغيلات التالية أسرع لأن النموذج محفوظ مسبقاً.
Python — استدعاء Ollama محلياً
Python · Ollama Local Call
import requests

# Ollama يعمل على المنفذ 11434 محلياً
response = requests.post(
    "http://localhost:11434/api/generate",
    json={
        "model": "llama3",
        "prompt": "اشرح ما هو الـ reverse shell بمصطلحات بسيطة.",
        "stream": False
    }
)

result = response.json()
print(result["response"])
⚡ الجزء الثاني — إعداد Groq API

Groq خدمة ذكاء اصطناعي سحابية مبنية على معالجات LPU (وحدات معالجة اللغة) المخصصة، مما يجعلها من أسرع منصات الاستدلال الموجودة. تستخدم نفس تنسيق API الخاص بـ OpenAI، لذا التكامل سهل جداً.

  • 1
    إنشاء حساب مجاني

    زر console.groq.com وسجّل بإيميلك. الخطة المجانية كافية جداً للتعلم والمشاريع الصغيرة.

  • 2
    توليد مفتاح API

    اذهب إلى API Keys في لوحة التحكم ← اضغط Create API Key ← أعطه اسماً ← انسخ المفتاح فوراً (يظهر مرة واحدة فقط).

  • 3
    تخزين المفتاح بأمان

    لا تضعه أبداً في الكود مباشرة. استخدم متغير بيئي أو ملف .env. راجع النمط الآمن أدناه.

  • 4
    تثبيت حزمة Python

    نفّذ: pip install groq — تتكفّل حزمة Groq الرسمية بالمصادقة وإعادة المحاولة والبث.

⚠
تحذير أمني مهم: لا تنشر مفاتيح API أبداً على GitHub. أضف .env إلى ملف .gitignore. تعامل مع مفاتيح API كأنها كلمات مرور — إذا سُرّبت، أعد توليدها فوراً.
Python — إعداد Groq API بشكل آمن
Python · Groq API — Secure Setup
# الخطوة 1: أنشئ ملف .env (لا ترفعه أبداً على GitHub)
# GROQ_API_KEY=gsk_مفتاحك_هنا

import os
from groq import Groq
from dotenv import load_dotenv

load_dotenv()  # تحميل ملف .env

client = Groq(
    api_key=os.getenv("GROQ_API_KEY")
)

# استدعاء بسيط
response = client.chat.completions.create(
    model="llama3-8b-8192",
    messages=[
        {
            "role": "user",
            "content": "ما هي المؤشرات الشائعة لهجوم التصيد الاحتيالي؟"
        }
    ]
)

print(response.choices[0].message.content)
💡
النماذج المتاحة على Groq: llama3-8b-8192، llama3-70b-8192، mixtral-8x7b-32768، gemma2-9b-it. النموذج 70b أذكى؛ النموذج 8b أسرع.

المميزات الرئيسية — ما الذي تحصل عليه

🔒

الذكاء الاصطناعي المحلي والخصوصية

مع Ollama، بياناتك لا تغادر جهازك أبداً. ضروري لتحليل سجلات الحوادث الحقيقية أو البيانات الشخصية أو معلومات التهديدات السرية.

⚡

استدلال سحابي فائق السرعة

تحقق معالجات LPU الخاصة بـ Groq سرعات توليد تصل إلى 10 أضعاف مقارنة بمزودي السحابة الاعتياديين — مثالي للتحليل الفوري.

🐍

تكامل Python السهل

لكلتا الأداتين واجهات Python نظيفة. ثبّت حزمة واحدة، اكتب 10 أسطر، احصل على ردود من الذكاء الاصطناعي. لا خبرة بالتعلم الآلي مطلوبة.

🛠️

بناء أدوات صغيرة

اصنع مساعدات سطر أوامر، محلّلات سجلات، مولّدات تقارير، أو أدوات فرز التنبيهات — كلها مدعومة بالذكاء الاصطناعي بكود بسيط.

💰

فعّال من حيث التكلفة

Ollama مجاني للأبد. Groq له خطة مجانية سخية. يمكنك بناء ونشر أدوات ذكاء اصطناعي حقيقية بتكلفة تكاد تكون صفراً.

🔌

متوافق مع OpenAI

Groq يتبع نفس تنسيق OpenAI API. إذا تعلّمت كيف تستدعي GPT، يمكنك استدعاء Groq — نفس الكود، عنوان مختلف.

حادثة أمنية نشاط مشبوه مكتشف — جهاز داخلي

يتلقى محلل أمني تنبيهاً عند الساعة 2:47 صباحاً. جهاز كمبيوتر داخلي يُولّد حركة مرور صادرة غير معتادة. لديه بيانات syslog خام، وقائمة عمليات، ووصف مختصر لسلوك برمجية خبيثة. يحتاج إلى الفرز السريع وإعداد تقييم أوّلي.

import requests

# مقتطف سجل مشبوه (يبقى على الجهاز المحلي)
suspicious_log = """
[2025-03-29 02:47:13] WARN: Process 'svchost.exe' spawned by 'explorer.exe'
[2025-03-29 02:47:14] NET: Outbound connection to 185.220.101.47:4444 (TOR exit node)
[2025-03-29 02:47:15] FS: File created at C:\\Temp\\payload32.exe
[2025-03-29 02:47:16] REG: HKLM\\Software\\Microsoft\\Run → payload32.exe added
[2025-03-29 02:47:18] PROC: cmd.exe /c whoami && net user && ipconfig /all
"""

system_prompt = "أنت مساعد محلل DFIR. حلّل سجلات الأمان وحدّد مؤشرات الاختراق (IOCs) وتقنيات MITRE ATT&CK والإجراءات الموصى بها."

user_message = f"حلّل هذه السجلات المشبوهة وقدّم: 1) ملخص ما حدث، 2) تقنيات MITRE ATT&CK، 3) الإجراءات الفورية الموصى بها:\n\n{suspicious_log}"

response = requests.post(
    "http://localhost:11434/api/generate",
    json={
        "model": "llama3",
        "prompt": f"[SYSTEM]: {system_prompt}\n\n[USER]: {user_message}",
        "stream": False
    }
)

analysis = response.json()["response"]
print("=== تحليل الذكاء الاصطناعي ===")
print(analysis)
Python — مولّد تقارير الحوادث باستخدام Groq API (سريع / سياق كبير)
Python · Incident Report — Groq API
from groq import Groq
import os
from dotenv import load_dotenv

load_dotenv()
client = Groq(api_key=os.getenv("GROQ_API_KEY"))

malware_behavior = """
ملف تنفيذي أُسقط في C:\\Temp. تحقّق من الاستمرارية عبر مفتاح التشغيل.
اتصل بعقدة خروج TOR معروفة على المنفذ 4444. أجرى استطلاعاً للنظام
(whoami, net user, ipconfig). احتمال إنشاء قناة C2.
"""

response = client.chat.completions.create(
    model="llama3-70b-8192",
    messages=[
        {
            "role": "system",
            "content": "أنت محلل استجابة للحوادث. أنشئ تقارير حوادث منظمة ومهنية باللغة العربية."
        },
        {
            "role": "user",
            "content": f"أنشئ تقرير حادثة رسمياً لهذا السلوك الخبيث:\n{malware_behavior}"
        }
    ]
)

report = response.choices[0].message.content
print(report)

تجربة مباشرة — جرّب الآن

تحاكي الأداة أدناه كيف يستجيب نموذج الذكاء الاصطناعي للاستفسارات الأمنية. اختر نوع الاستفسار، عدّل النص إذا أردت، واضغط تحليل لرؤية منطق الذكاء الاصطناعي.

Python — دالة استعلام ذكاء اصطناعي شاملة
Python · Universal AI Query Function
def query_ai(prompt, use_local=True, model="llama3"):
    """
    دالة شاملة: أرسل استفساراً إلى Ollama (محلي) أو Groq (سحابي).
    use_local=True  → Ollama (خاص، مجاني، يعمل بدون إنترنت)
    use_local=False → Groq  (سريع، سحابي، سياق أكبر)
    """
    if use_local:
        # Ollama المحلي
        response = requests.post(
            "http://localhost:11434/api/generate",
            json={"model": model, "prompt": prompt, "stream": False}
        )
        return response.json()["response"]
    else:
        # Groq السحابي
        client = Groq(api_key=os.getenv("GROQ_API_KEY"))
        resp = client.chat.completions.create(
            model="llama3-70b-8192",
            messages=[{"role": "user", "content": prompt}]
        )
        return resp.choices[0].message.content

# أمثلة الاستخدام:
result = query_ai("تحليل نشاط مشبوه",      use_local=True)
result = query_ai("إنشاء تقرير حادثة",       use_local=False)

حالات الاستخدام — تطبيقات حقيقية

🕵️

مساعد DFIR

أرسل للذكاء الاصطناعي الجداول الزمنية للحوادث. احصل على ملخصات فورية لما حدث ومتى وبأي ترتيب — دون حاجة للفرز اليدوي.

📋

محلّل السجلات

الصق مخرجات syslog أو جدار الحماية أو SIEM. اطلب من النموذج رصد الشذوذات وتحديد الأنماط المشبوهة واستخراج عناوين IP وهاشات الملفات.

📄

مولّد تقارير الحوادث

أدخل ملاحظاتك الخام والسجلات ومؤشرات الاختراق. احصل على تقرير حادثة منظم ومهني لإدارة أصحاب المصلحة.

🌐

مساعد استخبارات التهديدات

صف سلوك جهة تهديد. يربطها الذكاء الاصطناعي بمجموعات APT المعروفة والحملات الأخيرة ومصادر OSINT للتحقيق.

🗺️

ربط MITRE ATT&CK

صف السلوكيات الملاحظة للمهاجم. يقترح الذكاء الاصطناعي تلقائياً التكتيكات والتقنيات المطابقة في إطار ATT&CK (رموز T).

🔍

مستخرج مؤشرات الاختراق IOCs

الصق تقارير تهديدات أو بريداً إلكترونياً غير منظم. يستخرج الذكاء الاصطناعي عناوين IP والنطاقات والهاشات وأسماء الملفات وثغرات CVE.

🤖

روبوت فرز التنبيهات

أرسل تنبيهات SIEM إلى ذكاء اصطناعي يصنّفها: إيجابية حقيقية، إيجابية كاذبة، أو تحتاج مراجعة — مع تفسير منطقي لكل تصنيف.

🧰

مساعد كتابة الأدوات

اطلب من الذكاء الاصطناعي كتابة قواعد YARA أو Sigma أو محللات Python لصيغ سجلات مخصصة. سرّع تطوير الأدوات الأمنية.

  3. ابنِ سكريبت Python بسيط: اقرأ ملف سجل ← أرسله للذكاء الاصطناعي ← اطبع التحليل
  4. جرّب نماذج مختلفة (mistral، phi3، codellama) وقارن جودة تحليلها الأمني
  5. ابنِ أداة CLI صغيرة: أدخل وصف تنبيه ← احصل على ربط MITRE ATT&CK
  6. ادرس System Prompts — تعلّم كيف تمنح الذكاء الاصطناعي شخصية ثابتة للعمل الأمني
  7. استكشف Ollama API المتوافق مع OpenAI لاستبدال الأكواد الحالية بسهولة
